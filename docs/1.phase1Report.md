## Abstract

Global agricultural sustainability is increasingly threatened by crop diseases, especially in regions like Kerala where access to agronomic expertise is limited.

**NAVA (Next-gen Agricultural Virtual Assistant)** is proposed as an AI-driven system that integrates:

- **Computer Vision (CV)** → Disease detection  
- **Natural Language Processing (NLP)** → Advisory generation  

### Key Features:
- Multi-modal architecture  
- Superset dataset strategy (real-world robustness)  
- Deep learning + generative AI pipeline  

### Objective:
Provide scalable, mobile-accessible, expert-level agricultural guidance.

# Chapter 3: Methodology

## System Overview

NAVA uses a **dual pipeline**:
1. CNN → Disease detection  
2. LLM → Treatment generation  

---

## 3.1 Data Collection

### Superset Strategy:
Data aggregated from:
- PlantVillage  
- PlantWild  
- PlantDoc  
- PaddyDoctor  
- Kaggle datasets  

### Coverage:
- 34 disease classes  
- 7 crops  

---

## 3.2 Data Preprocessing

### Challenges:
- Severe class imbalance  

### Solution:
- **300–700 filtering rule**
- Data augmentation:
  - Rotation  
  - Brightness adjustment  
  - Blur simulation  

### Final Dataset:
- Training: 17,340  
- Validation: 3,060  
- Test: 4,089  

---

## 3.3 CV-NLP Pipeline

### Components:

- `app.py` → Interface controller  
- `cv.py` → Vision model  
- `nlp.py` → Language model  

### Workflow (from diagram on page 11):
1. User uploads image  
2. Preprocessing  
3. CV model → disease + confidence  
4. NLP model → treatment  
5. Final output  

---

## 3.4 Model Architectures

### ResNet50
- Deep residual network  
- Strong feature extraction  

### MobileNetV2
- Lightweight  
- Optimized for mobile  

### EfficientNet-B0
- Compound scaling  
- Best accuracy with fewer parameters  

---

## 3.5 Safety Mechanisms

- Confidence threshold = **0.85**
- If low confidence → no prescription  
- Rule-based fallback system  
- LLM generates detailed guidance

# Chapter 4: Results

## Model Comparison

| Model           | Accuracy | Training Time |
|----------------|---------|--------------|
| ResNet50       | 85.39%  | 5 min        |
| MobileNetV2    | 83.53%  | 4.5 min      |
| EfficientNet-B0| 94.54%  | 4.6 min      |

### Conclusion:
EfficientNet-B0 is the best performing model.
