# 1. Introduction

The agricultural sector is a cornerstone of the global economy but faces:

- 20–30% yield loss due to crop diseases  
- Lack of accessible agronomic expertise  
- Inefficiencies in manual diagnosis  

### Limitations of Existing AI Systems

- Poor performance in real-world conditions  
- Late-stage disease detection (RGB-only models)  
- Unsafe LLM-generated recommendations  
- Lack of explainability  

---

## Need for a New Approach

To overcome these challenges, systems must include:

- **Virtual Near-Infrared (VNIR)** → Early stress detection  
- **Explainable AI (XAI)** → Transparency  
- **Context-aware intelligence** → Environmental data fusion  
- **RAG-based LLMs** → Safe and factual recommendations  

---

## Proposed Solution

NAVA is a **Digital Agronomist Platform** integrating:

- EfficientNet-B0 (Computer Vision)  
- Llama 3.1 (NLP Advisory)  
- VNIR estimation  
- RAG-based knowledge system  
- Mobile-first deployment  

---

# 2. System Architecture Overview

NAVA is designed as a **modular multimodal system** consisting of four modules:

## Core Modules

| Module | Function |
|------|--------|
| **Mizhi** | Disease detection + VNIR monitoring |
| **Mozhi** | Conversational AI + memory |
| **Yukthi** | RAG + explainability |
| **Gathi** | Mobile app + backend orchestration |

---

### Architecture Insight (from diagram on page 5)

The architecture shows:

- User interacts via mobile app  
- Backend (FastAPI) orchestrates:
  - Diagnosis  
  - Chat  
  - RAG  
- Mizhi & Mozhi operate as parallel pipelines  
- Yukthi enhances both with knowledge + explainability  
- Gathi connects everything through mobile + API

## 3.1 Mizhi — Disease Detection & VNIR Monitoring

### A. Disease Detection (EfficientNet-B0)

#### Dataset
- Sources:
  - PlantVillage  
  - PlantWild  
  - PlantDoc  
  - PaddyDoctor  
  - Kaggle datasets  

#### Coverage
- 34 disease classes  
- 7 crops:
  - Rice, Corn, Tomato, Soybean, Cassava, Banana, Cucumber  

#### Preprocessing
- 300–700 samples per class  
- Data augmentation:
  - Rotation  
  - Brightness/contrast  
  - RGB shift  
  - Gaussian blur  

---

### Model Comparison

| Model | Accuracy | Training Time |
|------|--------|--------------|
| ResNet50 | 85.39% | 5 min |
| MobileNetV2 | 83.53% | 4.5 min |
| EfficientNet-B0 | **94.54%** | 4.6 min |

➡️ **EfficientNet-B0 selected**

---

### Key Features

- Confidence threshold: **0.85**
- Below threshold → **UNRELIABLE**
- Rule-based prescription system  
- ONNX deployment for:
  - Backend inference  
  - Mobile inference  

---

### B. Thanal — VNIR Estimation Engine

#### Dataset
- RGB–NIR paired dataset (capsicum leaves)
- Total: 1,615 image pairs  

---

### Architecture

- U-Net with attention gates  
- Input: 256×256  
- Output: Single-channel NIR  

---

### Training Details

- Loss: `0.85 L1 + 0.15 SSIM`  
- Optimizer: Adam  
- GPU: NVIDIA A100  

---

### Performance

- PSNR: **29.73 dB**  
- SSIM: **0.8939**  

---

### VNIR Monitoring Logic

#### Pipeline

1. HSV leaf segmentation:
   - Green → processed  
   - Yellow/Brown → critical  
   - None → skipped  

2. Compute:
```

NIR/Green Ratio

```

3. Baseline:
- First 5 scans  

4. Alert:
- If drop ≥ 15% → **STRESS ALERT**

---

### Additional Features

- Plant ID tracking  
- Daily monitoring notifications  
- Historical trend analysis

## 3.2 Mozhi — Conversational AI & Memory

### Core Functionality

- Llama 3.1 8B model  
- Persistent conversational memory  

---

### Hierarchical Memory System

| Level | Description |
|------|------------|
| Level 1 | Live session |
| Level 2 | Daily summary |
| Level 3 | Weekly digest |
| Level 4 | Season memory |

---

### Optimization Strategy

- Uses **Qwen2.5-1.5B** for summarization  
- Stores summaries in SQLite  

---

### Multilingual Support (Bhashini)

Pipeline:

```

Local Language → English → LLM → English → Local Language

```

Supports:
- Malayalam  
- Tamil  
- Hindi  

---

### Key Advantage

- Preserves domain-specific terminology  
- Enables farmer-friendly interaction

## 3.3 Yukthi — RAG + Explainable AI

### A. RAG Pipeline

#### Data Sources

- Kerala Agricultural University

---

### Pipeline

1. PDF ingestion

### Output

- Context-aware response  
- Source citations

# 4. System Workflow

---

## Disease Detection Flow

1. Capture leaf image  
2. EfficientNet → prediction  
3. Confidence check  
4. Grad-CAM visualization  
5. Rule-based prescription  
6. LLM-generated treatment  

---

## VNIR Monitoring Flow

1. Daily plant scan  
2. HSV filtering  
3. VNIR estimation  
4. Ratio computation  
5. Alert if stress detected  

---

## Conversational Flow

1. User query  
2. Retrieve:
   - Memory  
   - RAG data  
3. LLM generates response  
4. Translation (if needed)
